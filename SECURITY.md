# Security Policy

## Reporting a vulnerability

Do not open a public GitHub issue for security vulnerabilities.

Report vulnerabilities by email to **[security@openlabor.foundation](mailto:security@openlabor.foundation)**. Include a description of the issue, steps to reproduce, and the potential impact.

You can expect an acknowledgement within 48 hours and a resolution timeline within 7 days for confirmed vulnerabilities.

## Supported versions

These repositories are in an experimental state. Security fixes are applied to the current main branch only.

## Scope

- Vulnerabilities in source code in this repository
- Security issues in default configurations
- Hardcoded credentials or secrets (report immediately)

Out of scope: vulnerabilities in dependencies (report upstream), issues requiring physical access, social engineering.
